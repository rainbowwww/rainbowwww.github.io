http简介

Posted on 2018-04-26

###什么是HTTP？
全称为：超文本传输协议(Hyper Text Transfer Protocol)
两个电脑之间传输内容的协议,HTTP 的作用就是指导浏览器和服务器如何进行沟通。

###什么是curl？
>transfer a url

###HTTP的get请求示例：
1.用curl来示例：
用curl在gitbash上来创造一个请求：
curl -s -v -- https://www.baidu.com 得到请求结果如下：
> GET / HTTP/1.1 //动词 路径 协议/版本
> Host: www.baidu.com // Key1: value1
> User-Agent: curl/7.55.0 // Key2: value1
> Accept: */* // Key3: value1
> //换行
得到的响应如下：
< Accept-Ranges: bytes
< Cache-Control: private, no-cache, no-store, proxy-revalidate, no-transform
< Connection: Keep-Alive
< Content-Length: 2443
< Content-Type: text/html
< Date: Thu, 26 Apr 2018 08:00:23 GMT
< Etag: "588603e2-98b"
< Last-Modified: Mon, 23 Jan 2017 13:23:46 GMT
< Pragma: no-cache
< Server: bfe/1.0.8.18
< Set-Cookie: BDORZ=27315; max-age=86400; domain=.baidu.com; path=/
<
2.用 Chrome 查看请求 F12打开 Network
输入https://www.baidu.com 在 Network 点击，查看 request，点击「view source」
可以看到请求如下：
GET / HTTP/1.1
Host: www.baidu.com
Connection: keep-alive
Cache-Control: max-age=0
Upgrade-Insecure-Requests: 1
略……
可以看到响应如下：
HTTP/1.1 200 OK
Bdpagetype: 2
Bdqid: 0x9fcf424f0002e78a
Cache-Control: private
Connection: Keep-Alive
Content-Encoding: gzip

略……
总结get请求
由三个部分组成：
1 动词 路径 协议/版本
2 Key1: value1
2 Key2: value2
2 Key3: value3
3
>post 请求会多一个部分为要上传的数据。

1 协议/版本号 状态码 状态解释
2 Key1: value1
