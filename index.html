三次握爪和四次挥爪

Posted on 2019-04-14

三次握手

三次握手是客户端和服务端建立连接的过程。

过程如下：

  1. 客户端向服务器发送syn请求连接的信号。

  2. 服务器收到之后返回一个syn请求连接信号，ack和seq序号。ack是一方用来确认响应刚才发送请求的另一方。

  3. 客户端发送ack和seq序号来建立了连接。

总结：这是一个双方互相确认可以收取和发送的过程。客户端发送请求，服务器去响应，证明客户端有这个发送信息的能力，服务器有接收信息的能力。服务器在发送信息给客户端，客户端再去响应，证明服务器有这个发送信息的能力，客户端有这个接收信息的能力。

四次挥手

四次挥手是客户端和服务器断开连接的过程

过程如下 ：

  1. 主动方发送一个fin断开连接的请求信号和ack的报文并发送一个序号seq。
  2. 被动方接收到后发送ack并置发seq
  3. 被动方发送fin+ack并置发seq
  4. 主动方发送ack并置发seq

总结：主动方会发送信息说自己不再发送数据了，此时还是可以接收数据的，如果被动方没有发送完数据还可以接着发送数据，发送完数据后发起fin断开连接请求，然后主动方接收到后被动方关闭连接然后主动方断开。

正则的使用

处理字符串的常用方法

在说正则之前先说一下字符串的常用方法：

  1. search()

    字符串的查询

    如果有则返回所在位置，如果没有则返回-1

  2. subsrting()

    字符串的截取，有两个参数 ，开始位置和结束位置，包左不包右，也可以只给开始位置。

  3. charAt()

    返回指定位置的字符

  4. split()

    用于把一个字符串分割成字符串数组。

  5. join()

    可以把数组变成字符串

  6. match()

    可在字符串内检索指定的值，或找到一个或多个正则表达式的匹配。

还有很多，但当我们遇到需要进行匹配很多字符串的时候，这些方法就很鸡肋。所以我们需要正则

正则的使用

下面就说说正则的基础使用

  1. 正则的两种风格
    • new RegExp(‘a’) //这样则表示a本身 不经常使用
    • /a/ //使用频繁

  1. 参数

    • i 忽略大小写

    • g 全局匹配

  1. 转义

    • \d 数字

    • \w 英文 数字 下划线

    • \s 空白字符 包括空格 Tab 等

    • \D 除了字符串以外的数字

    • \W 除了英文数字下划线以外的东西

    • \S 除了空格和Tab以外的

  1. 量词
    • {n} 正好出现n次
    • {n,m}最少n次 最多m次
    • +是等价于{1, }最少有一次
    • ？等价于{0,1}最少0次最多一次
    • *等价于{0，} 可有可无的意思

JS里的数据类型：

JS里有七中数据类型分别是：number、 string、 boolean、 symbol、 undefined、 null、 object。除object为复杂类型，其余皆为简单类型。
复杂类型由简单类型组成。

number

整数和小数（比如1和3.14）
表达方式有：

  • 整数和小数：1 1.1 .1
  • 科学记数法：1.23e2
  • 二进制：0b11
  • 八进制：011（后来 ES5 添加了 0o11 语法）
  • 十六进制：0x11

string

字符串就是零个或多个排在一起的字符，放在单引号或双引号之中。

区分空格字符串和空字符串

  • 空格字符串长度为1
  • 空字符串长度为0

单引号(双引号)字符串的内部，使用单引号(双引号)，须在内部的单引号(双引号)前面加上反斜杠，用来转义。

单引号字符串的内部，可以使用双引号。双引号字符串的内部，可以使用单引号。

boolean

boolean 的取值
只有两个值：true 和 false
a && b 在 a 和 b 都为 true 时，取值为 true；否则为 false
a || b 在 a 和 b 都为 false 时，取值为 false；否则为 true

null和undefined

这两个值都表示什么也没有，其实是语言设计的一个bug。
两者的区分：
声明的变量没有值或非对象不想给值是undefined 对象object没有进行给值就是null

object

object 就是几种基本类型（无序地）组合在一起 object 的 key 一律是字符串，不存在其他类型的 key
object 里面可以有 object 如果object里面的key不加引号则key的命名需要遵循标识符的命名规则。
如果key的命名符合标识符的命名规则，则object[‘key’]可以写成object.key 注意 object.key 与 object[key] 不同
